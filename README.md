基于 FreeRTOS 的物联网环境监测与云上报系统
项目简介
本项目是一个基于 STM32F103 与 FreeRTOS 的多任务嵌入式系统，实现了本地多通道传感器数据采集、实时处理、异常报警以及通过 MQTT 协议将数据上报至 OneNET 云平台。系统充分利用 RTOS 的任务调度、消息队列、互斥锁等机制，解决了裸机开发中任务耦合度高、实时性难以保证的问题，并成功对接主流物联网云平台，具备远程监控与预警能力。

核心功能与技术亮点
1. 多任务实时调度
基于 FreeRTOS 构建 数据采集任务、OLED 显示任务、WiFi/MQTT 通信任务、异常报警任务，职责清晰，低耦合。

使用 消息队列 实现采集任务与显示/上传任务之间的数据传递。

使用 互斥锁 保护 OLED 显示等共享资源，避免资源竞争导致乱码。

2. 高效数据采集与软件滤波
ADC 采用 DMA 多通道扫描 模式，降低 CPU 负载。

对热敏电阻（温度）通道进行 滑动平均滤波，有效抑制采样噪声。

提供 NTC 热敏电阻（B 值公式）和 光敏电阻（Gamma 修正公式）到物理量（温度 °C、照度 Lux）的精确转换。

3. 物联网云接入（OneNET）
通过 ESP8266 (AT 指令集) 连接 WiFi，实现设备联网。

采用 MQTT 协议 与 OneNET 平台通信，支持设备认证、主题订阅与发布。

周期性上报 温度、光照度 数据至云端，同时订阅平台下发的控制指令（预留接口）。

实现了 MQTT 保活机制（keepalive = 120s），保证设备长期在线。

4. 异常报警机制
独立的高优先级报警任务，实时监测传感器数据。

当检测到温度/光照异常（或 ADC 值突变）时，通过 二值信号量 触发报警（LED 闪烁）。

报警阈值可灵活配置，未来可扩展为蜂鸣器或云端推送。

5. 内存安全与调试支持
监控 FreeRTOS 堆栈使用情况，防范内存泄漏。

支持通过串口打印系统运行状态、MQTT 连接日志等，便于现场调试。

硬件与开发环境
组件	型号/参数
主控芯片	STM32F103C8T6
WiFi 模块	ESP8266-01S (AT 固件 v2.3.0)
温度传感器	NTC 热敏电阻 (10kΩ, B=3950)
光照传感器	光敏电阻 (GL55 系列)
显示模块	0.96 寸 OLED (I2C 接口)
开发工具链	Keil MDK / VS Code + GCC
依赖库	STM32 HAL 库、FreeRTOS、CMSIS
云平台	中国移动 OneNET (MQTT 物联网套件)
