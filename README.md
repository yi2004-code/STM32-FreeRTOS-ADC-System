本项目是一个运行在 **STM32F103** 微控制器上的多任务嵌入式系统。结合 **FreeRTOS** 实时操作系统内核，实现了一套完整的“本地数据采集 → 软件滤波 → 异常报警 → 云端上报”物联网解决方案。

## 🎯 项目背景与解决痛点

在传统裸机开发中，数据采集、显示、通信、报警等功能往往混杂在同一个大循环（Super Loop）中，导致：
- **实时性差**：任务间相互阻塞。
- **维护困难**：代码耦合度高，难以扩展。
- **资源浪费**：无法充分利用 MCU 资源，多外设并发时容易丢数据。

本项目通过引入 FreeRTOS，将不同功能拆分为独立任务，利用**消息队列、互斥锁、信号量**等 IPC 机制实现高效协作，显著提升了系统的响应速度和稳定性。

## 🔧 核心功能

| 功能模块 | 具体实现 |
| :--- | :--- |
| 🌡️ **温度采集** | NTC 热敏电阻 + 分压电路，ADC 采样，B值公式转换为摄氏度 |
| 💡 **光照采集** | 光敏电阻 + Gamma 修正公式，转换为 Lux 照度值 |
| 📊 **软件滤波** | 滑动平均滤波（窗口大小 4），有效抑制采样噪声 |
| 🖥️ **本地显示** | 0.96 寸 OLED (I2C) 实时显示温度、照度 |
| ☁️ **云端上报** | ESP8266 通过 AT 指令连接 WiFi，MQTT 协议上报至 OneNET |
| 🚨 **异常报警** | 独立高优先级任务监测数据，超阈值触发 LED 闪烁 |
| 🔄 **任务通信** | 消息队列传递采集数据，互斥锁保护 OLED 共享资源 |

## 🧠 技术亮点

### 1. 多任务架构设计
系统划分为四个核心任务，职责单一，低耦合：
- **ADC_Task**: 负责传感器采集与滤波
- **Usart_Task**: 负责 OLED 显示刷新
- **Onenet_Task**: 负责 MQTT 协议栈处理与上报
- **Warning_Task**: 负责实时报警监测

### 2. 高效数据采集
- **DMA + ADC**: 采用 DMA 多通道扫描，CPU 负载极低，采集周期可短至 20ms。
- **精准转换算法**:
    - **温度**: Steinhart-Hart / B值公式，精度 ±0.5°C
    - **照度**: Gamma 校正公式，符合人眼感知特性

### 3. 稳健的云连接
- **协议**: MQTT 协议连接 OneNET，支持设备认证（Token）。
- **保活**: 实现 `keepalive = 120s` 机制，确保设备长期在线。
- **数据格式**: 上报格式严格符合 OneNET 物模型标准 (`$sys/.../thing/property/post`)。
- **容错处理**: 包含命令响应超时机制、错误重试与状态清理。

### 4. 实时报警机制
通过**二值信号量**触发，报警任务设为最高优先级，确保异常状态下能在毫秒级内响应。

## 📦 项目成果

- ✅ **稳定性**: 系统连续运行 72 小时以上无死机、无内存泄漏。
- ✅ **精度**: 温度测量误差 < 1°C (-20°C ~ 80°C)。
- ✅ **响应**: 报警响应延时 < 100ms。
- ✅ **联网**: MQTT 上报周期 5 秒，数据成功写入 OneNET，设备状态稳定在线。
