基于 FreeRTOS 的物联网环境监测与云上报系统
本项目是一个运行在 STM32F103 微控制器上的多任务嵌入式系统，结合 FreeRTOS 实时操作系统内核，完成了一套完整的本地数据采集 → 软件滤波 → 异常报警 → 云端上报的物联网解决方案。

🎯 项目背景与解决痛点
传统裸机开发中，数据采集、显示、通信、报警等功能往往混杂在同一个大循环中，导致：

任务间相互阻塞，实时性差

代码耦合度高，难以维护和扩展

无法充分利用 MCU 资源，多外设并发时容易丢数据

本项目通过引入 FreeRTOS 实时操作系统，将不同功能拆分为独立任务，利用消息队列、互斥锁、信号量等 IPC 机制实现任务间的高效协作，显著提升了系统的响应速度和稳定性。

🔧 核心功能
功能模块	具体实现
🌡️ 温度采集	NTC 热敏电阻 + 分压电路，ADC 采样，B值公式转换为摄氏度
💡 光照采集	光敏电阻 + Gamma 修正公式，转换为 Lux 照度值
📊 软件滤波	滑动平均滤波（窗口大小 4），有效抑制采样噪声
🖥️ 本地显示	0.96 寸 OLED (I2C) 实时显示温度、照度
☁️ 云端上报	ESP8266 通过 AT 指令连接 WiFi，MQTT 协议上报至 OneNET
🚨 异常报警	独立高优先级任务监测数据，超阈值触发 LED 闪烁
🔄 任务通信	消息队列传递采集数据，互斥锁保护 OLED 共享资源
🧠 技术亮点
多任务架构：划分为 ADC_Task（采集/滤波）、Usart_Task（OLED 显示）、Onenet_Task（MQTT 上报）、Warning_Task（报警监测），职责单一，低耦合。

高效数据采集：ADC 采用 DMA 多通道扫描，CPU 负载极低，采集周期可短至 20ms。

精准物理量转换：

温度：Steinhart-Hart / B值公式，精度 ±0.5°C

照度：Gamma 校正公式，符合人眼感知特性

云平台完整接入：

ESP8266 通过 AT 指令集配网

MQTT 协议连接 OneNET，支持设备认证（Token）

实现 keepalive = 120s 保活机制，设备长期在线

上报格式符合 OneNET 物模型标准 ($sys/.../thing/property/post)

稳健的通信处理：

命令响应超时机制

错误重试与状态清理

队列满保护（非阻塞发送，避免采集任务阻塞）

实时报警：通过二值信号量触发，报警任务优先级最高，确保异常状态毫秒级响应。

📦 项目成果
✅ 系统连续运行 72 小时以上无死机、无内存泄漏

✅ 温度测量范围：-20°C ~ 80°C，误差 < 1°C

✅ 照度测量范围：10 ~ 1000 Lux，满足室内环境监测需求

✅ MQTT 上报周期 5 秒，数据成功写入 OneNET，设备状态稳定在线

✅ 报警响应延时 < 100ms
